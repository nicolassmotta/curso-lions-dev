# Projeto Calculadora: Requisitos e Escopo

**Turma:** Lions Startups/LionsDev  
**Tópicos:** Definição de Requisitos, Escopo de Projeto, Critérios de Aceite e Planejamento de Software.

---

### 1. Objetivo do Projeto (visão geral)

Construir uma calculadora no terminal (Node.js) que:
- Ofereça um menu de operações matemáticas;
- Leia números do usuário com prompt;
- Valide entradas inválidas (ex.: texto onde se espera número);
- Mantenha um resultado acumulado entre operações;
- Permita encerrar o programa de forma explícita.

**Conceitos de aula envolvidos:** estruturas de controle (`if`/`switch`), loops (`while`), funções e validação de entrada.

### 2. Por que Definir Requisitos Antes de Codar?

Sem requisitos, "fazer uma calculadora" é tão vago quanto dizer **"vamos construir um carro"**: não sabemos se é popular, esportivo, elétrico, quantas portas, orçamento, etc.

Requisitos reduzem retrabalho: você sabe quando terminou e o que testar.

### 3. Escopo — Quais Operações Estão Dentro do Projeto?

Para **este** projeto da calculadora em aula, o escopo inclui:

- Adição
- Subtração
- Multiplicação
- Divisão
- Porcentagem (em relação ao valor acumulado, conforme implementação)
- Exibir o resultado atual
- Sair do programa

O que fica **fora do escopo** (a menos que você decida estender):
- Operações científicas (seno, log, potência com expoente variável);
- Histórico de todas as operações em arquivo;
- Interface gráfica (GUI).

### 4. Critérios de Aceite (como saber se está "certo")

Exemplos de critérios verificáveis:

- **a)** O programa exibe um menu numérico e aceita apenas opções válidas ou informa erro para opção inválida.
- **b)** Para cada operação que pede número, se o usuário digitar algo que não converte para número útil, o sistema avisa e não corrompe o estado.
- **c)** O laço principal só termina quando o usuário escolhe sair (ex.: opção equivalente a "Sair").
- **d)** Existe forma de ver o resultado acumulado sem precisar aplicar outra operação (ex.: item "Ver resultado").
- **e)** Divisão: idealmente tratar divisão por zero com mensagem clara (requisito de robustez).

### 5. Onde Está no Código?

Arquivo de referência (exercício em aula / slide):

`Modulo03/exercicios_slides/calculadora.js`

- Funções por operação: `somar`, `subtrair`, `multiplicar`, `dividir`, `porcentagem`
- Menu e loop: variável `op`, `while (op != 7)`, `console.log` do menu
- Decisão por opção: `switch (op) { case 1: ... }`
- Validação após entrada: `Number(prompt(...))` e `isNaN(num)`
- Resultado acumulado: variável `resultado`
