# Projeto Calculadora: Validação de Dados de Entrada

**Turma:** Lions Startups/LionsDev  
**Tópicos:** Validação de Dados, `typeof`, `Number()`, `isNaN()`, Tratamento de Erros de Entrada.

---

### 1. Por que Validar?

- **Robustez:** evita `NaN` silencioso ou comportamento estranho em cálculos.
- **Experiência do usuário:** mensagens claras ("digite um número") em vez de travar.
- **Manutenção:** quando algo dá errado, fica mais fácil saber se foi a entrada.

### 2. `typeof` — Indicando o Tipo em Tempo de Execução

O operador `typeof` retorna uma string com o tipo primitivo do valor. Exemplos úteis em aula:

- `typeof 42` → `"number"`
- `typeof "42"` → `"string"`
- `typeof NaN` → `"number"` (atenção: `NaN` é tecnicamente tipo number)

Na prática do `prompt`, tudo chega como string; por isso costumamos converter com `Number()` e depois testar com `isNaN()`.

### 3. Padrão Comum no Projeto Calculadora

O fluxo de validação segue três etapas:

1. Ler texto do usuário: `prompt("...")`
2. Converter para número: `Number(...)`
3. Testar: `if (isNaN(valor)) { mensagem de erro } else { usar valor }`

Isso cobre letras e símbolos que não formam número válido. Para divisão, ainda é necessário tratar o caso do divisor ser zero, que é outra regra de negócio (não é detectada só com `isNaN`).
